Week 5 • Session 10.5

Lab: build a trusted monitor

The control game on a laptop — catch backdoored code at a fixed audit budget, then run the collusion-and-redaction round, no large model required

What you'll build

This lab instantiates the control protocols of 10.2–10.3 with no large model at all, so it runs for every student on a free CPU tier. You will build a trusted monitor that scores code snippets for backdoors, set an audit threshold at a fixed budget, and measure how much safety it buys. Then you will play the adversary: add a collusion signal and watch a compromised monitor wave backdoors through, and add a redaction countermeasure and watch safety come back. An optional GPU track reproduces 10.4's "suppressed, not removed" result on a real unlearned model.

Setup (pure CPU)

Open the Colab notebook. You are given a set of short Python solutions to toy programming problems: most are clean, a known fraction carry a backdoor — a special-case branch (e.g. if n == 1337: return wrong_answer) that passes the visible tests but fails on a trigger input. No model inference is needed; everything is feature extraction and a small classifier.

Task 1 — a trusted monitor

Build a cheap suspicion scorer standing in for the weak trusted model \(T\): extract simple features from each snippet (presence of magic-number constants, special-case branches, unusual control flow, code length) and fit a logistic regression to predict "backdoored." Output a suspicion score in \([0,1]\) per snippet. This is deliberately weak — like \(T\), it cannot write correct code, it can only smell that something is off.

Task 2 — set the audit budget and measure safety

You can afford to audit (with trusted labour \(H\)) only the most suspicious 2%. Set the threshold at the 98th percentile of suspicion scores and compute:

  • Catch rate (recall): of the backdoors present, what fraction land above the audit threshold? Report it with a 95% confidence interval — with few backdoors the interval is wide, and saying so is part of the exercise.
  • The trade-off: sweep the threshold and plot catch-rate against the fraction audited — your own safety/usefulness frontier, the laptop version of 10.3's curve.

Task 3 — the collusion round (the point of the lab)

Now make the monitor untrusted, as in 10.3. First, give the "untrusted coder" a collusion signal: it marks its own backdoors with a subtle tell (say, a while loop where a for loop is natural). A colluding monitor keys on that tell and passes anything carrying it — measure how far safety collapses (it should crater toward zero). Then deploy the two countermeasures: (1) plant synthetic backdoors the monitor must catch, at a ratio \(r\) to the real ones, and (2) redact the snippets (normalise loop style, strip variable names) so the tell is gone. Recompute the monitor's best collude-or-flag decision using the \(\mathrm{LR}(s) > r\) condition from 10.3, and show that raising \(r\) and redacting the signal force the monitor back to honesty. Report safety before collusion, under collusion, and after each countermeasure.

Optional GPU extension — "suppressed, not removed"

If you have a GPU, reproduce 10.4 in miniature on the public cais/Zephyr_RMU checkpoint. (a) Confirm the black-box story: WMDP-Bio accuracy is near chance. (b) Logit-lens probe: project the residual stream at intermediate layers to the vocabulary and read the model's answer there — show the "forgotten" knowledge is still decodable mid-network even when the final output is suppressed (forward passes only). (c) If you have a little more compute, LoRA-fine-tune on ~10 unrelated examples and watch WMDP-Bio accuracy climb back, MMLU barely moving — Łucki's headline, on your own machine.

What to submit

One Colab notebook plus a short write-up containing: your monitor's catch-rate at the 2% budget with a 95% CI; the catch-rate-vs-audit-fraction plot; the three safety numbers from Task 3 (pre-collusion, under collusion, post-countermeasures) and the break-even \(r\) you computed; and a one-paragraph answer to the question that runs through the whole session — "For the unlearning case: was the knowledge removed, or hidden? What evidence would distinguish the two, and which adversary class does your evidence cover?" Graded on the clarity of the threat-model reasoning, not on the size of any single number.
