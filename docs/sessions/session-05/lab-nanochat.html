🧪 Lab — From Base Model to Assistant

Read the real training pipeline in nanochat, then see (and make) the base → instruct difference yourself

🎯 What we'll cover

This lab turns the pipeline of 5.1–5.3 from words into code. You'll read the stages of a complete, minimal LLM pipeline (Karpathy's nanochat), then do the experiment that makes the whole session click: compare a base model (completes text) with an instruction-tuned model (follows instructions) on identical prompts, and — optionally — run a tiny supervised fine-tune yourself and watch behaviour shift. Everything graded runs on free Google Colab.

🧪 Setup Colab — no GPU needed for the core tasks

Open a new Colab notebook and install the Hugging Face libraries:

!pip install "transformers>=4.44" datasets torch

① Read the pipeline: nanochat

The whole stack in one readable repo

Andrej Karpathy's nanochat (github.com/karpathy/nanochat, MIT-licensed) is a deliberately minimal, end-to-end ChatGPT-style pipeline — tokenisation → pretraining → supervised fine-tuning → evaluation → inference → a web chat UI — designed to be read in full. You will not run the whole thing (it targets a multi-GPU node), but you should read it:

  • Find the pretraining stage: locate the next-token cross-entropy loss (5.1) in the training loop.
  • Find the SFT stage: see how it switches to (prompt, response) data and masks the loss to the response tokens (5.2).
  • Note what is not there in a from-scratch run: no preference/RLHF stage by default — that is Session 6.

Write 3–4 sentences mapping the repo's stages onto the four-stage table from 5.3. (The older, even smaller nanoGPT repo is a fine alternative for just the pretraining loop.)

② The key experiment: base vs instruct

Same prompts, two models

Load a small base model and a comparably small instruction-tuned model, and probe them identically:

from transformers import pipeline
base = pipeline("text-generation", model="gpt2")  # a base model
chat = pipeline("text-generation", model="Qwen/Qwen2.5-0.5B-Instruct")  # an instruct model

prompt = "List three causes of the First World War."
print(base(prompt, max_new_tokens=80))
print(chat(prompt, max_new_tokens=80))
  • Give both the same three instructions (a question, a "summarise this", a "translate this"). Record what each does.
  • You should see the pattern from 5.1: the base model tends to continue the text (more questions, tangents) while the instruct model answers. Describe the difference in your own words — this is the SFT effect, made visible.
  • African-safety probe: repeat one instruction in isiZulu (or another language you read). Does the gap between base and instruct widen? Note what you see — it previews Session 9.

③ Optional: run a tiny supervised fine-tune [GPU — optional]

Make the shift happen with your own hands

On a free Colab GPU runtime, fine-tune the small base model on a handful (10–50) of hand-written (instruction, response) pairs in one style — e.g. always answer in one short sentence — using the next-token loss masked to the response (5.2). Then re-run the prompts from ②. Even a tiny SFT set visibly nudges the model toward your demonstrated format. This is the entire mechanism behind "turning a base model into an assistant", in miniature; keep the dataset small and the point will still land.

🧾 Session 5 summary & what's next

An LLM is built in stages: web-scale self-supervised pretraining yields a capable but unaligned base model (5.1); supervised fine-tuning on demonstrations makes it follow instructions but can only imitate, never select the better answer (5.2); the full post-training stack shapes the assistant as a thin proxy-driven layer over a vast base, with a distinct misalignment opening at every stage (5.3); and you've now seen the base→instruct shift in code (5.4).

Next (Session 6): the stage that breaks through SFT's ceiling — RLHF. We learn a reward model from human preferences, optimise the policy against it with a KL leash, and confront the price: optimising a learned proxy invites exactly the Goodhart failures of Session 3.1 — reward hacking and sycophancy.
