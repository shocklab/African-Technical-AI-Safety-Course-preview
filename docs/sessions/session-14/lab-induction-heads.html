Week 7 • Session 14

Lab: find and verify an induction head

See the induction stripe in TransformerLens, score the heads, and verify the mechanism by ablation

What you'll build

Session 13 derived the induction-head circuit on paper. This lab finds one in a real model and verifies it causally, using TransformerLens, a hooked reimplementation of GPT-style models that exposes every internal activation for caching and intervention. The lab is forward passes and ablations, no training, so the core runs on a free Colab CPU tier. You will turn the two detectors from 13.4 (prefix-matching and copying scores) into code, identify the heads, and show that knocking them out destroys in-context pattern completion, reproducing Olsson et al.'s ablation argument at toy scale.

Setup

Open the Colab notebook and install a pinned TransformerLens rather than the latest release. TransformerLens 3.x (v3.5.1 at the time of writing) deprecates the legacy HookedTransformer API this lab is written against and recommends TransformerBridge instead, so an unpinned pip install transformer_lens can hand you a library whose recommended API no longer matches these instructions. Pin a known-good 2.x release (a pip install transformer_lens==2.x style pin, with the version fixed to the one tested); instructor pre-flight: verify before class that the pinned version still loads both models below. Load two models on device="cpu": gpt2-small (124M; a real model with MLPs) and attn-only-2l (a 2-layer attention-only demo model where the maths of 13.3–13.4 is exact). The core idioms: logits, cache = model.run_with_cache(tokens) to capture activations, and model.run_with_hooks(tokens, fwd_hooks=[...]) to intervene.

Core tasks (CPU)

  • 1. Load and sanity-check. Tokenise a sentence, run with cache, confirm sensible next-token predictions. Inspect the cache keys so you know where attention patterns live (blocks.{l}.attn.hook_pattern).
  • 2. See the induction stripe. Build a repeated random sequence (a block of random tokens concatenated with an identical copy) and plot per-head attention patterns (e.g. with circuitsvis). In the second copy, look for the off-diagonal "stripe" where each token attends to the position after its first occurrence. That stripe is the induction head, visible to the eye.
  • 3. Find the previous-token head. On natural text, score every head by how much it attends to position \(i-1\). The top scorer in an early layer is head 1 of the circuit.
  • 4. Score induction heads. Implement the prefix-matching (induction) score from 13.4: on the repeated sequence, average each repeat-token's attention back to the token that followed its first occurrence. Rank all heads; the top scorer(s) are your induction head(s). Optionally implement the copying score from the eigenvalues of \(W_U W_{OV} W_E\).
  • 5. Verify by ablation (the causal step). With run_with_hooks, mean-ablate every top-scoring induction head from task 4: each one singly, then all of them together, and (separately) the previous-token head, measuring the rise in loss on the repeated-sequence task each time. Before running, write down a prediction: will a single-head ablation destroy the behaviour or only dent it? If the behaviour partly survives a single ablation, you have met backup heads, other heads picking up the ablated head's job, which 15.2's IOI circuit documents in detail. Ablating all the top scorers together, or the previous-token head, should destroy the induction behaviour, direct evidence that the two stages compose, as 13.3–13.4 predicted.

Stretch (optional, still CPU-feasible)

  • Direct logit attribution: decompose the correct-token logit into per-head contributions on the repeat; confirm the induction head dominates.
  • One layer vs two: run your detector on a 1-layer attention-only model and show induction is absent, empirically confirming 13.4's "needs two layers."
  • Mechanistic: inspect the induction head's \(W_{OV}\) eigenvalues (copying ⇒ positive), tying the detected head back to the 13.4 maths.

What to submit

One annotated Colab notebook containing: the induction-stripe attention plot; the ranked induction-score table with the identified head(s) named; and the ablation loss numbers (each induction head singly, all top scorers together, and the previous-token head), with your written prediction and one or two sentences linking the outcome to the 13.3/13.4 mechanism, including whether any single ablation left the behaviour partly intact. Graded on whether the causal story is established (does ablation break it?), not on polish.

Readings

Reference

• Olsson et al. (2022) (arXiv:2209.11895): the prefix-matching and copying definitions you are implementing, and the ablation argument you are reproducing.

• TransformerLens documentation (transformerlensorg.github.io/TransformerLens): run_with_cache, hook names, and ablation hooks; ARENA's Intro to Mechanistic Interpretability section (TransformerLens and induction circuits) is the fuller guided version.
